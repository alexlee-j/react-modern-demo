## 介绍

useRef 是 React 的一个 Hook，它提供了一个可变的 ref 对象，其 `.current` 属性被初始化为传入的参数。返回的 ref 对象在组件的整个生命周期内保持不变。

## 基本概念

### 1. 基本语法

```typescript
const refContainer = useRef<T>(initialValue);
```

- `T`: 可选的泛型类型参数
- `initialValue`: 初始值（可选）
- 返回一个可变的 ref 对象，其 `.current` 属性被初始化为传入的参数

### 2. 特点

- ref 对象在整个组件生命周期内保持不变
- 修改 ref.current 不会触发组件重新渲染
- 可以存储任意类型的值
- 常用于访问 DOM 元素

## 与 useState 的区别

### 1. 更新机制
- **useState**: 更新值会触发组件重新渲染
- **useRef**: 更新值不会触发重新渲染

### 2. 值的持久性
- **useState**: 每次渲染都会返回最新的状态值
- **useRef**: 在组件的整个生命周期中保持不变

### 3. 使用场景
- **useState**: 用于需要触发UI更新的状态
- **useRef**: 用于不需要触发重新渲染的值，如：
  - DOM 引用
  - 定时器 ID
  - 上一次的值
  - 计数器（不需要显示在UI上）

### 4. 更新时机
- **useState**: 异步更新，批量处理
- **useRef**: 同步更新，立即生效

## 使用场景

### 1. 访问 DOM 元素
最常见的用途，用于直接操作 DOM：

```typescript
const MyComponent = () => {
  const inputRef = useRef<HTMLInputElement>(null);
  
  const focusInput = () => {
    inputRef.current?.focus();
  };

  return (
    <input ref={inputRef} />
  );
};
```

### 2. 保存定时器 ID
清理定时器时特别有用：

```typescript
const MyComponent = () => {
  const timerRef = useRef<NodeJS.Timeout>();

  useEffect(() => {
    timerRef.current = setInterval(() => {
      // 定时器逻辑
    }, 1000);

    return () => {
      clearInterval(timerRef.current);
    };
  }, []);

  return null;
};
```

### 3. 记录前一次的值
配合 useEffect 使用：

```typescript
const MyComponent = () => {
  const [count, setCount] = useState(0);
  const prevCountRef = useRef<number>();

  useEffect(() => {
    prevCountRef.current = count;
  }, [count]);

  return (
    <div>
      现在: {count}, 之前: {prevCountRef.current}
    </div>
  );
};
```

## 最佳实践

### 1. 何时使用 useRef
✅ 适合使用 useRef 的场景：
- 需要直接访问 DOM 元素
- 需要存储不触发重新渲染的值
- 需要在整个生命周期中保持的值
- 需要在事件处理器中访问最新值

❌ 不适合使用 useRef 的场景：
- 需要在 UI 中显示的值
- 需要根据值的变化触发重新渲染
- 需要响应式的状态管理

### 2. 注意事项
1. 不要过度使用 useRef
   - useRef 虽然方便，但会使代码难以追踪
   - 优先考虑使用 useState 等声明式方案

2. 小心内存泄漏
   - 及时清理定时器和事件监听器
   - 在组件卸载时进行清理

3. TypeScript 类型标注
   - 为 ref 添加正确的类型
   - 处理可能为 null 的情况

## 常见问题

### Q1: 为什么我的 ref 值在渲染中没有更新？
A: ref.current 的更新不会触发重新渲染。如果需要在 UI 中显示，应该使用 useState。

### Q2: ref.current 可能是 null 吗？
A: 是的，特别是在访问 DOM 元素时。总是检查 null：
```typescript
if (inputRef.current) {
  inputRef.current.focus();
}
```

### Q3: 可以在渲染期间更新 ref 吗？
A: 不建议。ref 更新应该在事件处理器或 effects 中进行。

### Q4: useRef 和 createRef 有什么区别？
A: useRef 在组件的整个生命周期中返回相同的 ref 对象，而 createRef 每次渲染都会创建一个新的 ref。
