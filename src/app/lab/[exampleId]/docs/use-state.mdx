# useState 钩子

## 为什么需要 useState？

在 React 中，普通变量在组件重新渲染时会被重置。而 `useState` 提供了一种方式来保存组件的状态数据，使其在重新渲染时保持不变。

```jsx
// ❌ 错误方式：使用普通变量
let count = 0;
function Counter() {
  // count 在每次渲染时都会被重置为 0
  return <button onClick={() => count++}>{count}</button>;
}

// ✅ 正确方式：使用 useState
function Counter() {
  const [count, setCount] = useState(0);
  // count 的值会被 React 保存，不会在重新渲染时丢失
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

## 核心概念

useState 是 React 中用于在函数组件中添加状态的 Hook。它允许你在组件中声明状态变量，并提供一个函数来更新这些变量。每次状态更新时，React 会重新渲染组件以反映最新的状态。

### 工作原理

1. **声明状态**：`useState` 接收一个初始值，返回一个数组
2. **读取状态**：第一个数组元素是当前状态值
3. **更新状态**：第二个数组元素是更新函数
4. **重新渲染**：当使用更新函数时，React 会重新渲染组件

## 实现步骤

```jsx live=true
function Counter() {
  // 声明一个新的状态变量，称为 "count"
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>你点击了 {count} 次</p>
      <button onClick={() => setCount(count + 1)}>
        点击我
      </button>
    </div>
  );
}
```

## 使用说明

- `useState` 返回一个数组，我们使用数组解构来获取：
  1. 第一个元素：当前状态值
  2. 第二个元素：更新状态的函数
- 可以在组件中多次调用 `useState` 来管理多个状态变量
- 状态更新函数可以接收新值或函数作为参数

## 进阶用法

### 1. 惰性初始化

如果初始状态需要复杂计算，可以传递一个函数：

```jsx
// ❌ 每次渲染都会执行复杂计算
const [state, setState] = useState(complexCalculation());

// ✅ 只在首次渲染时执行复杂计算
const [state, setState] = useState(() => complexCalculation());
```

### 2. 批量更新状态

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  const handleClick = () => {
    // ❌ 这样写只会加1，因为这些更新会被批处理
    setCount(count + 1);
    setCount(count + 1);
    
    // ✅ 使用函数式更新来正确地增加2
    setCount(c => c + 1);
    setCount(c => c + 1);
  };

  return <button onClick={handleClick}>{count}</button>;
}
```

### 3. 对象状态的更新

```jsx
const [user, setUser] = useState({ name: '', age: 0 });

// ❌ 错误：这会完全替换状态对象
setUser({ name: 'Alice' }); // age 字段会丢失

// ✅ 正确：使用展开运算符保留其他字段
setUser(prev => ({ ...prev, name: 'Alice' }));
```

## 注意事项

1. **状态更新是异步的**
```jsx
const [count, setCount] = useState(0);

// ❌ 可能不会得到预期结果
setCount(count + 1);
console.log(count); // 仍然是旧值

// ✅ 如果需要在更新后执行操作，使用 useEffect
useEffect(() => {
  console.log(count); // 会显示更新后的值
}, [count]);
```

2. **避免重复状态**
```jsx
// ❌ 不好：派生状态应该通过计算得到
const [firstName, setFirstName] = useState('');
const [lastName, setLastName] = useState('');
const [fullName, setFullName] = useState(''); // 不需要！

// ✅ 好：通过计算得到派生状态
const [firstName, setFirstName] = useState('');
const [lastName, setLastName] = useState('');
const fullName = `${firstName} ${lastName}`; // 计算得到
```

## 最佳实践

1. 使用有意义的变量名
```jsx
const [isLoading, setIsLoading] = useState(false);  // ✅ 好
const [x, setX] = useState(false);                  // ❌ 差
```

2. 合理拆分状态
```jsx
// ✅ 好：相关状态分开管理
const [firstName, setFirstName] = useState('');
const [lastName, setLastName] = useState('');

// ❌ 差：过度组合可能导致不必要的重渲染
const [user, setUser] = useState({ firstName: '', lastName: '' });
```

3. 使用函数式更新
```jsx
// ✅ 好：基于之前的状态更新
setCount(prevCount => prevCount + 1);

// ❌ 差：可能导致竞态条件
setCount(count + 1);
```
